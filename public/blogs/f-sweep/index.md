# F-Sweep：给容器界面加一个顺手按钮

创建时间：2026-02-28 01:09:14 UTC  
仓库：<https://github.com/wwwqqq001/F-Sweep>  
状态：私有仓库

F-Sweep 是一个 Minecraft 1.21.1 NeoForge 模组。

它解决的是一个很具体的游戏体验问题：打开容器时，把可存入的物品快速转入 Beyond Dimensions 网络或相关存储，而不是手动一格一格整理。

![F-Sweep 容器交互流程图](/blogs/f-sweep/interaction.svg)

这张图把一次操作拆成了客户端和服务端两侧：玩家打开容器后，按钮或快捷键只负责发起请求；真正的槽位判断、物品转移和结果确认放在服务端完成；最后再把反馈显示到界面右上角。这样可以避免客户端“看起来能点”，但服务端状态不一致的问题。

F-Sweep 的价值不在功能名有多大，而在边界补得够细。图示能让读者看到一个小按钮背后仍然包含界面注入、快捷键、网络包、槽位过滤、服务端验证和用户反馈几个环节。

## 功能目标

我希望这个模组足够轻：

- 不改变游戏核心流程。
- 不增加复杂配置。
- 只在需要整理物品的时候出现。
- 操作后给出清晰反馈。

最后形成的体验是：打开容器，点击按钮或按下快捷键 `F`，可转移物品自动处理。

## 核心能力

项目 README 里记录了几个主要能力：

- 容器界面添加“一键存入”按钮。
- 支持快捷键 `F`。
- 自动跳过玩家背包界面和不应转移的槽位。
- 屏幕右上角显示操作结果。

这些能力看起来简单，但实现时要考虑客户端界面、服务端逻辑、网络包、槽位策略和提示文案。

## 技术点

Minecraft 模组开发里，最容易出问题的是“看起来能动，但边界不安全”。

F-Sweep 需要处理：

- 哪些界面能触发。
- 哪些槽位不能转移。
- 客户端按钮如何发请求。
- 服务端如何验证请求。
- 存储满了如何反馈。
- 没有可转移物品时如何反馈。

这个项目适合写成案例，因为它不是炫技，而是围绕一个小体验把边界补齐。

## 和 Beyond Storage Addon All 的关系

F-Sweep 更像单版本功能实现；`beyond_storage_addon_all` 更像多版本发布管理。

一个负责“功能怎么做”，一个负责“多个版本怎么长期发”。
