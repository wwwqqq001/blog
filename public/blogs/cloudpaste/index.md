# CloudPaste：一次 Cloudflare 自托管实践

创建时间：2026-05-15 09:53:42 UTC  
仓库：<https://github.com/wwwqqq001/CloudPaste>  
状态：公开 fork

CloudPaste 是一个 Serverless 文件管理与 Markdown 分享工具。这个仓库是 fork，我把它放进作品集，不是为了说它完全由我从零实现，而是记录一次 Cloudflare 自托管工具的研究和实践。

## 为什么关注它

我想要的是一个轻量的文件管理工具：

- 能管理文件。
- 能分享文本和 Markdown。
- 能接不同存储。
- 能部署在 Cloudflare 或 Docker。
- 最好不需要自己维护传统服务器。

CloudPaste 的方向很接近这个需求。

## 能力拆解

它比较有意思的地方在于把多个能力放在一起：

- Cloudflare Workers 提供运行入口。
- D1 可以保存元数据。
- R2 或 S3 兼容存储可以保存文件。
- WebDAV 让文件管理能接入更多客户端。
- Markdown 编辑器让文本分享不只是“传一个 txt”。
- 文件预览覆盖图片、视频、PDF、Office、代码等格式。

## 我关注的工程点

### 多存储抽象

一个文件工具如果只能绑定单一存储，很快会受限。多存储聚合的意义是把上传、预览、分享、挂载这些动作抽象出来，让底层可以是 R2、S3、WebDAV 或其它服务。

### Cloudflare 限制

Cloudflare 很适合轻量服务，但也有请求体、执行时长、流量和运行时限制。真正部署这类工具时，要关注上传方式、分片、直传和预签名 URL。

### 前后端部署方式

CloudPaste 支持前后端一体，也能拆分部署。对个人项目来说，一体化部署省心；对长期维护来说，拆分更容易扩展。

## 这篇文章的定位

这不是原创项目介绍，而是自托管实践记录。

如果以后我基于它做自己的文件管理方案，重点会放在：

- 更简单的个人配置。
- 更清晰的存储接入。
- 更安全的分享权限。
- 更适合移动端的管理体验。
